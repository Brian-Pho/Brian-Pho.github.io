ENSF 480: Bug Tracking System (Nov 2017)

Source Code

This project was done in a group of three where the goal was to create a bug tracking system using a software engineering design methodology. Due to the nature of being in school, we only had time to do a simple linear development process which has an analysis, design, and implementation phase. The analysis phase included creating a use-case diagram, sequence diagram, and activity diagram for obtaining a high level overview of the functional requirements and constraints. The design phase then moved onto more detailed diagrams such as a UML class diagram, deployment diagram, and state transition diagram for a bug. Lastly, the project was implemented in Java where Git and IntelliJ were used to code the project.

The project is a system used by developers and managers for tracking software bugs reported by users. A user is able to submit a bug report that is then approved by the manager. Once the manager approves/disapproves of the bug, it is then assigned to a developer to be fixed. When the bug is fixed, the developer marks the bug as fixed and carries on. This project did not require the use of a database so a simple text file was used.

From this project I learned how a software engineer might go through the process of developing a good piece of software. It was interesting how the initial analysis and design phases made it easier to implement the program afterwards as most of the thinking had been done during those two phases. The implementation just required converting the diagrams into code which isn't too bad. I had some fun times during the project such as implementing a singleton design pattern for the login and creating the GUIs.

CPSC 471: Library Database Management System (June 2017)

Source Code

While this project was part of my database class, it can be thought of as a continuation and combination of my last two projects. This project was done in a group of three and I was assigned to do the UI aspect of the app. For this project, my group had to create an app that connected to a database (mySQL) and retrieve data from the database using SQL queries. This project was challenging because the project required the use of multiple programming languages (Java, SQL, PHP, and XML) and required the use of an external server to host the data.

The purpose of this project was to allow library users access to the library database from the app. Their are two different users, users and admins, with admins having additional functionalities. Their were a lot of functions that we had to incorporate such as search, add, delete, and update which increased the complexity of the project. Overall, this project was very challenging but I learned a lot from it.
