ENSF 480: Bug Tracking System (Nov 2017)

Source Code

The goal of this project was to build a system to store and track software bugs. A user is able to submit a bug report that is then approved by a manager. Once the manager approves/disapproves of the bug, it's then assigned to a developer to be fixed. When the bug is fixed, the developer marks the bug as fixed and carries on.

From this project, I learned how a software engineer might go through the process of developing a good piece of software. It was interesting how the initial analysis and design phases made it easier to implement the program afterwards as most of the thinking had been done during those two phases. The implementation required converting various diagrams into code which wasn't too bad. I had some fun times during the project such as implementing a singleton design pattern for the login and creating the GUIs.

CPSC 471: Library App (June 2017)

Source Code

This project was done in a group of three and I was assigned to do the UI aspect of the app. My group had to create an app that connected to a database (mySQL) and retrieved data from the database using SQL queries. This project was challenging because the project required the use of multiple programming languages (Java, SQL, PHP, and XML) and required the use of an external server to host the data.

The purpose of this project was to allow library users access to the library database via the app. There are two different users: normal users and admins. There were a lot of functions that we had to incorporate such as search, add, delete, and update which increased the complexity of the project. Overall, this project was very challenging but I learned a lot from it.

Source Code
